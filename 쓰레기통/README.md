# 📚 프로젝트 구조 문서 (Project Structure Documentation)

> **🎯 목적:** AI가 프로젝트를 효율적으로 관리하고, 자동으로 구조를 유지하도록 돕는 문서 시스템

**생성일:** 2026-01-23  
**버전:** 1.0.0

---

## 🌟 개요

이 폴더는 **프로젝트의 두뇌**입니다. AI(Cursor)가 코드를 작성할 때 항상 참조해야 하는 핵심 문서들이 들어있습니다.

### 왜 필요한가?
```
❌ 문서 없이 AI에게 명령
   → AI가 프로젝트 구조를 모름
   → 잘못된 위치에 코드 작성
   → 리팩토링 필요

✅ 문서를 참조하도록 명령
   → AI가 정확한 위치 파악
   → 올바른 파일에 코드 작성
   → 자동으로 깔끔한 구조 유지
```

## 📁 문서 구조

### 1. **ARCHITECTURE.md** ⭐⭐⭐⭐⭐
```
역할: 프로젝트 전체 구조의 실시간 스냅샷
용도: AI가 프로젝트 구조를 파악할 때 사용

내용:
├── 디렉토리 구조 (전체 파일 트리)
├── 기능별 파일 매핑
├── 모듈 의존성 그래프
├── 데이터베이스 스키마
├── CSS 변수 시스템
└── Storage 버킷 정보

언제 읽나요?
- 새 기능 추가 전
- 파일 위치 확인할 때
- 의존성 확인할 때
- 전체 구조 파악할 때
```

### 2. **FILE_MAP.md** ⭐⭐⭐⭐⭐
```
역할: 파일별 상세 정보 (함수, 변수, 의존성)
용도: AI가 특정 함수/파일을 찾을 때 사용

내용:
├── 각 파일의 역할
├── export 함수 목록
├── 함수별 상세 정보 (입력, 출력, 용도)
├── 전역 변수 목록
├── 의존성 정보
└── 마지막 수정 날짜

언제 읽나요?
- 특정 함수 수정할 때
- 함수 위치 찾을 때
- 함수 사용법 확인할 때
- 의존성 확인할 때
```

## 🚀 빠른 시작 가이드

### Step 1: 문서 읽기 (처음 한 번만)
```
1. 이 README.md 읽기 (지금!)
2. ARCHITECTURE.md 훑어보기 (전체 구조 파악)
3. AI_USAGE_GUIDE.md 읽기 (명령 방법 학습)
```

### Step 2: AI에게 명령하기
```
항상 이렇게 시작:

"@프로젝트 구조/ARCHITECTURE.md 를 참고해서 [작업 내용]"
```

### Step 3: 문서 자동 업데이트
```
명령 끝에 항상 추가:

"완료 후 ARCHITECTURE.md와 FILE_MAP.md를 업데이트해줘"
```

## 💡 사용 예시

### 예시 1: 새 기능 추가
```
사용자 명령:
"@프로젝트 구조/ARCHITECTURE.md 를 참고해서 좋아요 기능을 추가해줘.

요구사항:
- 작품 상세보기에 좋아요 버튼
- 좋아요 수 표시
- 본인이 좋아요 했는지 표시

완료 후 ARCHITECTURE.md와 FILE_MAP.md를 업데이트해줘."

AI 작업:
1. ARCHITECTURE.md 읽기 → 구조 파악
2. 적절한 파일 위치 결정 (js/likes.js)
3. 코드 작성
4. ARCHITECTURE.md 업데이트 (파일 추가, 의존성 그래프)
5. FILE_MAP.md 업데이트 (함수 목록 추가)
```

### 예시 2: 버그 수정
```
사용자 명령:
"@프로젝트 구조/FILE_MAP.md 에서 uploadPost 함수를 찾아서 수정해줘.

버그: 이미지 업로드 실패 시 에러 메시지가 표시 안 됨

완료 후 FILE_MAP.md에 수정 내용 기록해줘."

AI 작업:
1. FILE_MAP.md 읽기 → uploadPost 위치 확인 (js/upload.js)
2. 함수 수정
3. FILE_MAP.md 업데이트 (마지막 수정 날짜, 변경 내용)
```

## ❓ 자주 묻는 질문 (FAQ)

### Q1: 문서를 항상 참조해야 하나요?
```
A: 네! 문서 참조 없이 명령하면 AI가 프로젝트 구조를 모르고
   잘못된 위치에 코드를 작성할 수 있습니다.
   
   ✅ "@프로젝트 구조/ARCHITECTURE.md 를 참고해서..."
```

### Q2: 문서 업데이트를 잊어버리면?
```
A: 문서가 오래되면 AI가 잘못된 정보를 참조하게 됩니다.
   명령 끝에 항상 업데이트 요청을 추가하세요.
   
   ✅ "완료 후 ARCHITECTURE.md와 FILE_MAP.md를 업데이트해줘"
```

### Q3: 어떤 문서를 먼저 읽어야 하나요?
```
A: 처음 시작:
   1. README.md (이 파일)
   2. ARCHITECTURE.md (전체 구조)
   3. AI_USAGE_GUIDE.md (명령 방법)
   
   개발 중:
   - 새 기능: ARCHITECTURE.md
   - 함수 수정: FILE_MAP.md
   - 규칙 확인: CODING_RULES.md
```

### Q4: 문서가 너무 길어지면?
```
A: 괜찮습니다! AI는 긴 문서도 빠르게 읽을 수 있습니다.
   오히려 상세할수록 더 정확하게 작업합니다.
```

### Q5: 문서 없이도 작동하나요?
```
A: 작동은 하지만 비효율적입니다.
   
   문서 없이:
   - AI가 구조를 모름
   - 잘못된 위치에 코드 작성
   - 리팩토링 필요
   - 시간 낭비
   
   문서 사용:
   - AI가 구조를 정확히 파악
   - 올바른 위치에 코드 작성
   - 깔끔한 구조 유지
   - 시간 절약
```
